Demo & documentation

October 21, 2017 by zxorn

To give you a demonstration of how Scrollan works and how you can use it, we've setup this Bootstrap 4 blog template (based on this template) as an example.

Try scrolling around on this page and you'll see how Scrollan will render a progress indicator under the top menu and a headings menu to the left. While you read through the instructions on this page you'll notice how Scrollan helps you keep track of where you are in the content structure.

Usage

As stated in the presentation of this component, to make things easier you can either run Scrollan as a jQuery plugin or without a framework using regular Javascript. There are no downsides to either method, just pick whatever method you are most comfortable with! To start out you need to:

  • Have a div where Scrollan can render it's content
  • Include the Scrollan CSS file
  • Include the Scrollan Javascript file

Let's use this page as an example, that way you can view the source to get the whole picture.

Using jQuery

To initialize Scrollan as a jQuery plugin we begin by selecting the div where we want Scrollan to render it's content: 

$( document ).ready(function() {
  $('#scrollan-output').Scrollan();
});

By not given Scrollan any options the default behaviour will be used:

  • Prefix all CSS classes with scrollan-
  • Use all headings (h1 to h5) on the current page
  • Use no top padding

To change this behaviour we can use 3 options:

Name Type Description
cssPrefix String Prefix to use for all CSS classes
headings jQuery object Selection of all headings to use
topPadding Number Amount of pixels to pad calculations

To give you an idea how those options are used, here's how you would initialize Scrollan on this demo page: 

$( document ).ready(function() {
  $('#scrollan-output').Scrollan({
    cssPrefix: 'scrollan',
    headings: $('#blog-content').find(':header'),
    topPadding: $('#navbar-container').height()
  });
});

Without a framework

To initialize Scrollan on this page without a framework we begin by selecting the output div and then initialize Scrollan: 

document.addEventListener('DOMContentLoaded', function(event) {
  var scrollan = new Scrollan(
    document.querySelector('#scrollan-output')
  );
});

By not given Scrollan any more arguments the default behaviour will be used:

  • Prefix all CSS classes with scrollan-
  • Use all headings (h1 to h5) on the current page
  • Use no top padding

To change this behaviour we can use 3 extra arguments of the Scrollan function. Here are all arguments for Scrollan (including the output div):

Argument Type Description
1 HTMLElement Div to render Scrollan inside
2 String Prefix to use for all CSS classes
3 NodeList Selection of all headings to use
4 Number Amount of pixels to pad calculations

To give you an idea how those options are used, here's how you would initialize Scrollan on this demo page: 

document.addEventListener('DOMContentLoaded', function(event) {
  var headings = document
      .querySelector('#blog-content')
      .querySelectorAll('h1,h2,h3,h4,h5');

  var navbarHeight = document
      .querySelector('#navbar-container')
      .getBoundingClientRect()
      .height;

  var scrollan = new Scrollan(
    document.querySelector('#scrollan-output'),
    'scrollan',
    headings,
    navbarHeight
  );
});

Performance

Since the scrolling event can fire very frequently Scrollan only runs its internal update functionality when the browser is ready to update the screen. This is done by using the function window.requestAnimationFrame. Here's a quote about what this function does:

The window.requestAnimationFrame() method tells the browser that you wish to perform an animation and requests that the browser call a specified function to update an animation before the next repaint. The method takes a callback as an argument to be invoked before the repaint.

source

Scrollan doesn't do anything unless a scroll event has fired, this is to avoid using up resources on while no ones using the site. For example when the site is in an inactive browser tab.

Scrollan is based on Elm which has one of the fastest virtual DOM implementations around.

Styling

To allow easy styling a CSS class is assigned to each of the parts that make up the component along with the chosen CSS prefix. In the examples below the default CSS prefix scrollan is used.

There are 2 major parts of the component: the progress bar and the headings menu.

Progress bar

The progress bar is rendered by Scrollan each time you scroll. Here's what the output looks like on this demo page: 

<div class="scrollan-progress" style="width: 100%;">
  <div class="scrollan-progress-indicator"
       style="height: 100%; width: 13.1049%;">
  </div>
</div>

On this demo page the progress bar has a fixed position under the top menu with a width that stretches across the entire page, like so: 

/*
On this page we've put Scrollan inside the top menu so that we can put the
progress indicator at the bottom of the top menu no matter what height it has.
*/
.scrollan-progress {
  position: absolute;
  left: 0;
  bottom: 0;
  width: 100%;
  height: 0.2rem;
  z-index: 1030;
  background-color: #e9ecef;
}


.scrollan-progress-indicator {
  background-color: #43c995;
}

Headings menu

The headings menu is rendered as a hierarchy by Scrollan each time you scroll. Here's what the output looks like on this demo page: 

<div class="scrollan-menu">
  <div class="scrollan-menu-item">
    <a href="javascript:void(0)"></a>
    <div class="scrollan-menu-item">
      <a href="javascript:void(0)">Demo &amp; instructions</a>
      <div class="scrollan-menu-item">
        <a href="javascript:void(0)">Usage</a>
        <div class="scrollan-menu-item">
          <a href="javascript:void(0)">Using jQuery</a>
        </div>
        <div class="scrollan-menu-item">
          <a href="javascript:void(0)">Without a framework</a>
        </div>
      </div>
      <div class="scrollan-menu-item">
        <a href="javascript:void(0)">Performance</a>
      </div>
      <div class="scrollan-menu-item">
        <a href="javascript:void(0)">Styling</a>
        <div class="scrollan-menu-item scrollan-menu-item-active">
          <a href="javascript:void(0)">Progress bar</a>
        </div>
        <div class="scrollan-menu-item">
          <a href="javascript:void(0)">Headings menu</a>
        </div>
      </div>
    </div>
  </div>
</div>

On this demo page the headings menu has a fixed position to the left under the progress bar with padding for each level, like this: 

.scrollan-menu {
  position: fixed;
  top: 3.7rem;
  left: 0;
  width: 20%;
  z-index: 1030;
}


.scrollan-menu-item {
  padding-top: 5px;
  padding-left: 20px;
}
/* Root item in the menu should not have any padding */
.scrollan-menu > .scrollan-menu-item {
  padding-left: 0;
}
/* Currently active heading should show it's link as bold  */
.scrollan-menu-item-active > a {
  font-weight: bold;
}

The main content area and right side menu has been wrapped in a div with the class name content-wrapper so that the margin to the left can be controlled. Depending on what size of device the page is viewed on the visibility of Scrollan menu is also controlled. 

.content-wrapper {
  width: 70%;
  margin-left: 20%;
}


/*
Hide scrollan on devices smaller than Bootstraps "Large" breakpoint.

Read more about the available breakpoints here:
https://getbootstrap.com/docs/4.0/layout/overview/#responsive-breakpoints
*/
@media (max-width: 991px) {
  .scrollan-menu {
    display: none;
  }

  .content-wrapper {
    width: 100%;
    margin-left: 0;
  }
}
